## Общая инструкция по запуску всех проектов
Для запуска серверной части (backend) используется команда `dotnet run` в консоли в папке проекта (веб или консольного). 
При этом клиентская часть (браузерная, frontend) запускается следующим образом:
1. Запустить серверную часть.
2. Открыть в браузере адрес сервера (`localhost:порт`, обычно `8080`). Если в строке `app.Run` в файле `Program.cs` запускаемого проекта явно указан порт, используйте его. Если нет, то может быть назначен один из портов в диапазоне 5000–5003+. Например, `localhost:8080` или `localhost:8080/index.html` (дописывать `index.html` необязательно).
3. Обычно открывается страница, которая соответствует файлу клиента `wwwroot/index.html` — это и есть клиентская часть.
4. В некоторых проектах (`m007/Server`, `Masha\ChatServer`) внутри находится несколько клиентов, и у каждого свой HTML-вход и свой адрес. Для каждого из них нужно вручную написать в адресной строке соответствующее название.
    1. Например, для `m007/Server`:
        1. `localhost:8080/index.html` (`localhost:8080/`) — изменённая копия клиентской части **XAMIC\Serv «Хамик»**;
        2. `localhost:8080/sozvon/index.html` (`localhost:8080/sozvon/`) — изменённая копия клиентской части веб-проекта **NKrandZ\SignalRChat «Созвон»**;
        3. `localhost:8080/indexAndrey.html` — изменённая копия клиентской части веб-проекта **Papka2 «Скам-мессенджер»**;
        4. `localhost:8080/MegaLosos.html` — изменённая копия клиентской части веб-проекта **MegaLosos\SignalRChat «Мегалосось»**;
        5. `localhost:8080/YourSchedule/schedule.html` — изменённая копия клиентской части веб-проекта **Masha\ChatServer «планировщик»**;
    2. Для `Masha\ChatServer`:
        1. `localhost:8080/schedule.html` — планировщик.
